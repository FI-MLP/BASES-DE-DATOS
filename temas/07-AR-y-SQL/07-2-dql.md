## 7.1 Select

La instrucción select se emplea para obtener datos almacenados en la base de datos o cómo comúnmente se dice realizar consultas a la base de datos. Por eso algunos autores clasifican a esta instrucción como una categoría del lenguaje SQL.

Generalmente una consulta requiere de condiciones, joins, operaciones para hacer filtros, etc.

### 7.1.1 Sintaxis SQL estándar

```sql
select [distinct]
{
[<nametable>.]<column-name>| * |<expression>
| <column> as <alias>
},...
from
{
<table-or-view-name> | [[as] <table-alias>]
}
[where <predicate>]
[group by <column-name>,...
[having <predicate>]
]
[order by {<column-name>|<column-number>}
[asc|desc] ];
```

### 7.2.2 Rango

#### Cláusula BETWEEN...AND

Comprueba si un valor se encuentra entre dos valores especificados, utilizando la cláusula where.

**EJEMPLO 3.** Obtener a todos los empleados cuyo apellido paterno este entre BAENA y FLORES

```sql
SELECT * FROM empleadosEjemplo
where paterno between 'BAENA' and 'FLORES'
```

#### AND

Comprueba si un valor se encuentra entre dos valores especificados.

**Ejemplo Rango 4:** Obtenga el nombre de los empleados, sueldo, el puesto y ciudad de aquellos que tienen un salario --entre 12000 y 28000

```sql
SELECT nombre+' '+paterno+' '+materno as nombreEmpleado, sueldo, puesto, ciudad
FROM empleadosEjemplo
WHERE sueldo >= 12000 AND sueldo <= 28000
```

```sql
SELECT nombre+' '+paterno+' '+isnull(materno, '-SIN MATERNO-') as nombreEmpleado, sueldo, puesto, ciudad
FROM empleadosEjemplo
WHERE sueldo >= 12000 AND sueldo <= 28000
```

### 7.2.3 Pertenencia a un conjunto

#### Cláusula IN (va1, val2, val3, ...)

Comprueba si un valor dado coincide con uno de una lista de valores especificada.

**Ejemplo 5:** Nombre de los empleados, sueldo y el puesto de aquellos que ganan 2975, 35000 o 28000.

```sql
Select nombre+' '+paterno+' '+isnull(materno, '-') as nombreEmp, sueldo, puesto
From empleadosEjemplo
where sueldo in ( 2975 , 35000 , 28000 )
order by sueldo
```

### 7.2.4 Correspondencia con patrón

#### Cláusula LIKE

Obtiene las filas para las que el contenido de una columna de texto coincide con un patrón especificado. En este tipo de consulta se utiliza el símbolo `%` como comodín. También se utiliza el símbolo subrayado bajo (`_`), que representa exactamente una posición de carácter, sólo un carácter.

**Ejemplo 6:** Liste la clave y nombre del empleado, de aquellos cuyo nombre inicie con la letra D

```sql
SELECT clave, nombre+' '+paterno+' '+isnull (materno, '-') as nombre
FROM empleadosEjemplo
WHERE nombre LIKE 'D%';
```

**Ejemplo 7:** Muestre todos los datos de los empleados donde en la TERCERA posición de su apellido paterno tengan una letra A.

```sql
SELECT clave, nombre+' '+paterno+' '+isnull (materno, '-') as nombre
FROM empleadosEjemplo
WHERE paterno LIKE '__A%';
```

### 7.2.5 Contraste de valor nulo.

#### Cláusula IS NULL o IS NOT NULL

Comprueba si existen valores nulos en las columnas de una tabla.

**EJEMPLO 8:** Muestre la clave, nombre y sueldo de los empleados sin apellido materno.

```sql
SELECT clave, nombre, paterno, materno, sueldo
FROM empleadosEjemplo
WHERE materno IS NULL;
```

### 7.2.6 Contrastes compuestos.

#### Cláusulas AND, OR y NOT

Utilizando las reglas de la lógica se pueden combinar condiciones de búsqueda SQL simples para formar condiciones compuestas.

**EJEMPLO 9:** Liste el nombre de los empleados que son de honorarios y líder de proyecto o su sueldo es mayor o igual a $35000

```sql
SELECT nombre+' '+paterno+' '+isnull(materno, '-') as nombreEmp, puesto,
sueldo, tipo as 'TIPO EMPLEADO'
FROM empleadosEjemplo
WHERE TIPO='H' AND (sueldo >= 35000 or puesto='LIDER DE PROYECTO')
```

De la consulta anterior para el tipo muestre Honorarios y Confianza no H o C (CASE)

```sql
SELECT nombre+' '+paterno+' '+isnull(materno, '-') as nombreEmp, puesto,
sueldo,
(case
When tipo='H' then 'HONORARIOS'
When TIPO='C' then 'CONFIANZA'
end) as 'TIPO EMPLEADO'
FROM empleadosEjemplo
WHERE TIPO='H' AND (sueldo >= 35000 or puesto='LIDER DE PROYECTO')
order by sueldo
```

## 7.3 Condiciones de ordenamiento

#### Order by

Ordena los registros resultantes de una consulta por un campo o unos campos especificados en orden ascendente o descendente. Su sintaxis es:

```sql
SELECT <lista de campos>
FROM <tabla>
WHERE <criterios de selección >
[ORDER BY <campo1 [asc|des]>, [,campo2 [asc|des]]];
```

**Ejemplo 10.** Nombre de los empleados ordenados alfabéticamente en forma descendente.

```sql
SELECT paterno, materno, nombre
FROM empleadosEjemplo
ORDER BY paterno DESC, materno desc, nombre desc;
```

**Ejemplo 11.** Obtenga nombre completo en un solo campo, puesto, sueldo y tipo de empleado ordenados por sueldo de mayor de menor y puesto en orden alfabético de manera descendiente

```sql
SELECT nombre+' '+paterno+' '+isnull(materno, '-') as nombreEmp, puesto, sueldo,
(case
When tipo='H' then 'HONORARIOS'
WHEN TIPO='C' then 'CONFIANZA'
end) as 'TIPO EMPLEADO'
FROM empleadosEjemplo
order by sueldo desc, puesto desc
```

## 7.4 Distinct

La cláusula DISTINCT nos devuelve valores únicos. En una tabla, una columna puede contener valores duplicados; y algunas veces sólo se necesita un listado de los valores diferentes.

La cláusula DISTINCT se añade a las sentencias SELECT, justo después de la palabra clave SELECT.

**Ejemplo 11 (b):** Obtener la lista de los puestos sin repeticiones

```sql
SELECT DISTINCT puesto
FROM empleadosEjemplo;
```

**Ejemplo 12:** Obtener la lista de ciudades de donde habitan los empleados

```sql
SELECT DISTINCT ciudad
FROM empleadosEjemplo;
```

## 7.5 Alias en columnas y tablas

Un alias permite nombrar de una forma distinta a un campo, únicamente para ser presentada como parte de la respuesta de la instrucción SELECT.

**Ejemplo 13:** Obtener para cada empleado el total ganado (sueldo + aguinaldo), solo de aquellos empleados con valores distintos de NULL

```sql
select paterno, materno, nombre, sueldo+aguinaldo
from empleadosEjemplo
where sueldo is not null and aguinaldo is not null
```

Observar que en la sentencia anterior el nombre de la columna dice (no column name). En lugar de esto, es más adecuado nombrar esta columna con un valor más descriptivo

```sql
select paterno, materno, nombre, sueldo+aguinaldo as total
from empleadosEjemplo
where sueldo is not null and aguinaldo is not null
```

En sqlserver la palabra as es opcional, la siguiente consulta es equivalente

```sql
select paterno, materno, nombre, sueldo+aguinaldo total
from empleadosEjemplo
where sueldo is not null and aguinaldo is not null
```

Al igual que las columnas, se puede usar alias en el nombre de las tablas, esto es muy útil para los joins. En este caso normalmente las tablas se representan con unos cuantos caracteres para evitar escribir el nombre completo de la tabla. Este tema se revisará más adelante.

Cuando en una sentencia select intervienen más de una tabla, y si ambas tablas contienen columnas con el mismo nombre, (como en el ejemplo anterior), es necesario indicar en la sentencia select el nombre de la tabla del campo que queremos seleccionar empleando la sintaxis.

## 7.6 Operadores lógicos

Utilizados para evaluar un conjunto de condiciones, comprueban la veracidad de alguna condición. Así como los operadores de comparación, el resultado es un valor booleano, TRUE o FALSE.

| OPERADOR | DESCRIPCIÓN |
| :--- | :--- |
| **ALL** | TRUE si el conjunto completo de comparaciones evalúa a TRUE |
| **AND** | TRUE si ambas expresiones son verdaderas |
| **ANY** | TRUE si cualquier miembro del conjunto de comparaciones es TRUE |
| **SOME** | TRUE si alguna(s) de las comparaciones del conjunto evalúa a TRUE |
| **BETWEEN** | TRUE si el operando está dentro del intervalo |
| **EXISTS** | TRUE si una subconsulta regresa algún valor (contiene al menos un registro) |
| **IN** | TRUE si el operando es igual a algún valor de una lista de valores o expresiones |
| **LIKE** | TRUE si el operando coincide con algún patrón |
| **NOT** | Negación de un operador booleano. Invierte el valor de cualquier otro operador boleano |
| **OR** | Evalúa a true si alguna expresión booleana es verdadera |

#### ANY, SOME

Comparan un valor de columna o un valor literal con el resultado de una subconsulta que devuelve valores de una sola columna.

* Una subconsulta utilizada con el operador ANY o SOME solo puede devolver valores de una sola columna.
* El operador ANY o SOME debe estar precedido por operadores de comparación como `=`, `!=`, `>`, `>=`, `<`, `<=`.

**Ejemplo 14**

```sql
SELECT * FROM empleadosEjemplo
WHERE SUELDO < ALL (SELECT SUELDO FROM catalogo.puesto)
```

```sql
SELECT * FROM empleadosEjemplo
WHERE AGUINALDO < SOME (SELECT SUELDO FROM catalogo.puesto)
```

```sql
SELECT * FROM empleadosEjemplo
WHERE AGUINALDO < ANY (SELECT SUELDO FROM catalogo.puesto)
```

El operador ANY o SOME se puede utilizar con una expresión escalar. Por ejemplo, lo siguiente se utiliza con IF y una condición:

**Ejemplo 15:**

```sql
IF 15000 <= ANY(SELECT SUELDO FROM empleadosEjemplo)
print('Los salarios de algunos empleados son iguales o superiores a $15,000');
else
print('Los salarios de algunos empleados son menores a $15,000');
```

```sql
IF 15000 <= all(SELECT SUELDO FROM empleadosEjemplo)
print('Los salarios de todos empleados son iguales o superiores a $15,000');
else
print('Los salarios de los empleados son menores a $15,000');
```

#### EXISTS

El operador EXISTS se utiliza para crear condiciones booleanas para verificar si una subconsulta devuelve fila(s) o un conjunto vacío. Cuando su subconsulta devuelve al menos una fila, EXISTS devuelve TRUE. No importa cuántas filas se devuelvan ni cuántas columnas haya en la lista de la subconsulta SELECT. Sólo cuando la subconsulta devuelve 0 filas, EXISTS devuelve FALSE. Es importante señalar que ninguna de las filas devueltas por la subconsulta se muestra en el resultado final.

```sql
SELECT columns
FROM table
WHERE EXISTS (SELECT columns FROM table2);
```

**Ejemplo 16.** Obtenga todos los puestos del catálogo de puestos que existan en la tabla ejemplo

```sql
select * from catalogo.puesto p
where exists (select distinct puesto from empleadosEjemplo e
where p.descripcion=e.puesto )
```

### 7.7.1 Funciones de agregación

Las funciones de agregación generan valores de resumen en las consultas; procesan todos los valores seleccionados en una única columna para generar un único resultado.

Estas funciones se pueden aplicar a todas las filas de una tabla, a un conjunto de una tabla especificado por una cláusula WHERE o a uno o varios grupos de filas de una tabla. Cuando se aplica una función de agregación se genera un valor individual por cada conjunto de filas.

| Función | Acción |
| :--- | :--- |
| **COUNT(*)** | Regresa el número de registros encontrados |
| **COUNT(<campo>)** | Regresa el número de registros cuyo valor del campo especificado no es nulo |
| **SUM(<campo>)** | Suma los valores de la columna especificada |
| **AVG(<campo>)** | Promedia los valores del campo especificado |
| **MIN(<campo>)** | Regresa el valor mínimo del campo especificado |
| **MAX(<campo>)** | Regresa el valor máximo del campo especificado |

Ejemplos:

**17. Cantidad de empleados que son de confianza**
```sql
SELECT COUNT(clave)
FROM empleadosEjemplo
WHERE tipo='C';
```

**18. Monto total pagado**
```sql
select sum(sueldo) from empleadosEjemplo
```

**19. El sueldo máximo pagado a un empleado.**
```sql
SELECT MAX(sueldo)
FROM empleadosEjemplo ;
```

**20. Sueldo promedio de los empleados**
```sql
SELECT AVG(sueldo)
FROM empleadosEjemplo ;
```

**21. Monto que se paga a los empleados que no tuvieron aguinaldo**
```sql
select sum(sueldo) from empleadosEjemplo
where aguinaldo is not null
```

**22. Cantidad de empleados que recibieron aguinaldo, total de los sueldos de dichos empleados, así como el sueldo promedio**
```sql
SELECT COUNT(aguinaldo) AS 'CON AGUINALDO', SUM(SUELDO) as totalSueldo,
AVG(sueldo) promedio
FROM empleadosEjemplo ;
```

## 7.8 Funciones de agrupación

### 7.8.1 GROUP BY

En la cláusula GROUP BY se indica el o los campos por los cuales se desea agrupar un conjunto de registros. Comúnmente esta agrupación va acompañada con una serie de funciones agregación en la lista de columnas de la instrucción select. Como su nombre lo indica, GROUP BY se aplica para realizar cálculos sobre grupos de datos. Estos cálculos los realiza una función de agregación.

Considere que la lista de columnas de la cláusula group by corresponde a todas las columnas que no usan una función de agregación en la lista del select.

**EJEMPLO 23, 24**

Monto total necesario para pagar a los empleados de cada puesto.
```sql
SELECT puesto, SUM(sueldo)
FROM empleadosEjemplo
GROUP BY puesto;
```

Cuanto se gasta cada puesto en sueldo y aguinaldo
```sql
SELECT puesto, SUM(aguinaldo) AS 'SUMA AGUINALDO', SUM(sueldo) AS 'SUMA SUELDO'
FROM empleadosEjemplo
GROUP BY puesto;
```

### 7.8.2 GROUP BY...HAVING

Esta cláusula es el equivalente a la cláusula WHERE, es decir, especifica un criterio o condición, pero la diferencia radica en que se ocupa únicamente cuando se desea especificar una condición después de que se haya realizado una agrupación.

* `where` opera sobre columnas de los registros de una tabla
* `having` opera sobre el resultado que se produce al emplear group by.

**HAVING** es esa parte de tu consulta SQL que te permite decir: "Oye, solo muéstrame los grupos que cumplan con esta condición específica".

¿Cómo funciona?: Digamos que tienes una tabla de ventas y quieres saber qué vendedores han vendido más de cierta cantidad. Primero, agrupas las ventas por vendedor y luego usas HAVING para filtrar solo aquellos que han vendido, digamos, más de 10,000.

<img width="365" height="238" alt="Screenshot_27" src="https://github.com/user-attachments/assets/7c3a862f-8624-48cb-8f3e-08626545186d" />

**Ejemplo 25:**
```sql
SELECT ID_Vendedor, SUM(Monto) as TotalVentas
FROM Ventas
GROUP BY ID_Vendedor
HAVING SUM(Monto) > 10000 ;
```

**EJEMPLO 26.** Genere una sentencia que muestre el total del aguinaldo proporcionado a los empleados tester clasificados en base al tipo de empleado

```sql
select tipo, puesto, sum(aguinaldo)
from empleadosEjemplo
group by tipo, puesto
having puesto='TESTER'
```

**EJEMPLO 27:** Obtener el monto total de aguinaldo otorgado por puesto con aguinaldo mayor de 150000

```sql
select puesto, sum(aguinaldo) total
from empleadosEjemplo
group by puesto
having sum(aguinaldo)> 150000
```

## 7.9 Joins

Para realizar la consulta de información que está almacenada en varias tablas, es necesario relacionar, dicho en otras palabras, unir, ligar, asociar de alguna manera estas tablas, es decir, hacer join.

La asociación de las tablas se realiza igualando os valores de los campos que tengan en común, es decir, usando la Pk y la Fk de las tablas involucradas. Si retomamos la base de datos que estamos trabajando en clase, las tablas, empelado y familiar, el atributo en común es idempleado, es decir, las tablas están asociadas de manera directa a través de la FK en la tabla familiar.

### 7.9.1 Clasificación de joins

* Inner joins
    * Natural join
    * Inner join
* Outer join
    * Left outer join
    * Right outer join
* Cross join
* Self join

### 7.9.2 INNER JOIN

Combina registros de dos o más tablas siempre que existan valores coincidentes en un campo en común.

Los inner join únicamente incluyen en la salida de la consulta los registros que tengan correspondencia con la tabla asociada.

#### Equi Join, Non Equi Join.

Estos 2 conceptos se emplean únicamente para distinguir el tipo de operador que se emplea en la condición de un inner join.

* **Equi Join** emplea el operador de igualdad `=`. Por ejemplo, `e.idempleado=f.idempleado`. De lo anterior, los ejemplos vistos hasta el momento son Inner Joins tipo Equi Join ya que hacen uso del operador de igualdad.
* **Non equi join** emplea un operador diferente al de igualdad: `>`, `<`, `<=`, `>=`. Por ejemplo, `e.idempleado>=f.idempleado`. Son utilizados con muy poca frecuencia en la práctica.

#### Natural Join

Es una variante de inner join, el predicado se construye automáticamente por lo que puede ser omitido, en un inner join se debe especificar el predicado (condición). Sin embargo, en SQLServer natural join no existe.

#### Outer join

En un **outer Join** se obtienen los registros que tienen correspondencia en ambas tablas, similar a un inner join, y adicionalmente se obtienen los registros que no tienen una correspondencia en la otra tabla. Los campos que no contengan dicha correspondencia se presentan como nulos. Existen 2 casos de outer joins: left outer join y right outer join.

**Sintaxis estándar:**

### 7.9.7 Joins con múltiples tablas.

El resultado de hacer un join entre 2 tablas se puede visualizar como una nueva tabla virtual, a la cual se le puede aplicar otro join con la siguiente tabla indicada en la sentencia haciendo una especie de encadenamiento con la instrucción join.

**Ejemplo:**
Seleccionar el nombre, apellidos y puesto de los empleados que trabajan en el proyecto 13

```sql
select nombre, paterno, materno, descripcion as puesto, id_proyecto
from trabajador.EMPLEADO e
inner join catalogo.PUESTO p on e.id_puestoActual=p.idpuesto
inner join proyecto.empleado_proyecto ep on e.idempleado=ep.idempleado
where id_proyecto= 13
```

### 7.9.8 Self Joins

En un self join una tabla está relacionada consigo misma en lugar de ser relacionada con otra. Internamente se trabaja con 2 conjuntos de datos, ambos asociados a la misma tabla.

No existe como tal una cláusula para este tipo de Join, es decir, no existe la palabra 'self'.

**Ejemplo:**
Mostrar el nombre y apellidos de los empleados, así como el nombre y apellidos de su jefe, el cual también es un empleado.

```sql
select e.idEmpleado, e.nombre, e.paterno, e.materno, e.idEmpleado_jefe,
j.nombre nomJefe, j.paterno patJefe, j.materno matJefe
from trabajador.EMPLEADO e
left join trabajador.EMPLEADO j on j.idempleado=e.idEmpleado_jefe
```

En este tipo de Joins se debe tener cuidado en el uso de alias y en la condición del join. Se emplea la misma tabla 2 veces, pero con alias diferentes. El alias 'e' identifica a los empleados, y el alias 'j' a su jefe.

En la condición del Join, se debe igualar el identificador del supervisor de un empleado con su identificador del empleado. De hacer otra combinación se obtendrán resultados incorrectos.

Otra forma de construir el predicado es pensar en ambos conjuntos como tablas separadas. Id_empleadojefe es la FK que proviene de una tabla padre con llave primaria id_empleado. En este caso la PK de la tabla padre representa a los jefes, y es id_empleado. Por lo tanto, el predicado se forma igualando la PK de la tabla padre (j.id_empleado) con la FK de la tabla hija (e.id_empleadojefe). Es decir, `e.id_Empleadojefe=j.id_Empleado`
